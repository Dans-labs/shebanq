{{=T('Get started')}}

You can read passages and encounter other people's published queries:
  • jump to a book and chapter via browse
  • while reading, click get related queries
  • highlight the query results

{{=T('Next steps')}}

You can contribute interesting queries yourself:
  • register/login
  • follow the My Queries menu item and add a query
  • or select an existing one for editing
  • type an MQL search (consult the {{=A('MQL quick reference guide', _target='_blank',_href=URL('static', 'docs/MQL-QuickRef.pdf'))}} and {{=A('ETCBC4 transcription table', _target='_blank',_href=URL('static', 'docs/ETCBC4-transcription.pdf'))}} )
  • execute the query
You can (un)publish your queries:
  • go to a query through My Queries and then the edit button
  • press the Publish or Make Private button
When you press Publish the query will be executed first, in order to make sure that the query results match the query instruction. Published queries show up under Public Queries. They also are shown in the margin when you browse Bible chapters containing their results.

{{=T('About')}}

This is shebanq: the first data service of ancient-data.org. Shebanq is a search engine for the Hebrew Bible, powered by the ETCBC4 linguistic database, formerly know as WIVU. The data is archived for open access, clicking the DANS logo above will bring you to the treasure trove. You can read about the history leading to this application by clicking on SHEBANQ in the menu bar. Moreover, the source of this web app is on Github.

{{=T('History')}}

2014-07-29
SHEBANQ goes live, but it is a demonstrator, with open issues. Currently we are aware of:
  1. Publishing queries is not as smooth as it should be
  2. The query engine might cause exceptions that are not yet handled
  3. The rendering of the text is not completely correct. Webkit browsers (Chrome, Safari) are known to mangle white space around accented Hebrew characters, and even Firefox has errors. Some of these are caused by the interaction between plain UNICODE Hebrew text and surrounding HTML <span> elements. Others occur even inside plain UNICODE text. If you work on a computer, we recommend Firefox. But we have tried very hard to work around the issues with Chrome and Safari, so that you can enjoy the otherwise beautiful typography also on your iPad. However, there might be problems that we have not spotted before. Feedback welcome.
