About

SHEBANQ is a search engine for the Hebrew Bible, powered by the ETCBC4 linguistic database, formerly know as WIVU. The data is archived for open access, and the program code is open source. A list of sources needed to recreate all underlying data processing is included below.

Project

SHEBANQ started as a project with the same name, funded by CLARIN-NL, the Dutch section of the European Common Language Resources Infrastructure project CLARIN.

It was carried out from mid 2013 to mid 2014 by the Eep Talstra Centre for Bible and Computer ETCBC and Data Archiving and Networked Services DANS, an institute of the Royal Netherlands Academy of Arts and Sciences KNAW and the Netherlands Organization for Scientific Research NWO.

DANS and the ETCBC, partnered with the German Bible Society DBG and the Nederlands Bible Society, NBG contribute to the sustained availability of the texts and data in SHEBANQ.

License and Re-use

The creators of the linguistic annotations that drive SHEBANQ, the ETCBC, want to facilitate optimal access to their data by researchers. All operations that are needed for the scholarly process are allowed. You are allowed to download the data and the code, build your own workflows on top of the data and even to redistribute your versions of the data and the code.

The ETCBC prefers to stress the potential of using its data, rather than the legal restrictions in doing so. It has published data, tools and documentation online that enables teams of biblical scholars and developers to reproduce all data processing that is needed for SHEBANQ. The recommended tool to start with is {{=A('laf-fabric', _href='https://github.com/ETCBC/laf-fabric', _target='_blank')}}. Its documentation describes in detail how to download the data and set up data-processing workflows. For an example of what you can do with the LAF-Fabric/SHEBANQ combination, see below.

The ETCBC welcomes contributions that improve and enrich the interface of SHEBANQ. We are also open to implement mutual deep linking with other biblical data sources.

ETCBC
The ETCBC organization on Github has several repositories related to its Hebrew Text Database.
SHEBANQ
This is the code that drives the SHEBANQ website. It is based on web2py, whose code is also on Github.
shebanq-doc
Feature-by-feature documentation of the Hebrew Text Database. Here you find the meaning of the linguistic annotations to the text. This documentation is written in rst (= ReStructured Text), and from this html documentation can be built. The built html is included in SHEBANQ.
LAF-Fabric
The Hebrew Text Database is archived in the LAF format. LAF-Fabric is a data preprocessing tool that can handle LAF. It is a Python package. On top of it there are packages specialized in the peculiarities of the ETCBC data.
LAF-Fabric notebooks
This a repository full of executable notebooks, showing a wide range of things that can be accomplished. A recommended way to do data processing with the ETCBC data is using the concept of the IPython Notebook. In a notebook you can describe, document and execute Python code. Such notebooks can easily be shared through github and NBviewer.
EMDROS
The engine that performs MQL queries against an Emdros text database. Written in C++, by Ulrik Sandborg-Petersen.

User generated content

Users are an important source of data for SHEBANQ. They may contribute queries that highlight research questions and solutions, and they may contribute notes to individual clauses in the text.

Contributing note sets

Notes are the preferred way to add observations and comments that are not carved in stone. They can also be used to view the outcomes of algorithms operating on the Hebrew Bible in context. As an example, here is a set of notes about applying verbal valence patterns to occurrences of selected Hebrew verbs. The notes have been created by means of LAF-Fabric in this tool and from there they have been bulk-uploaded to SHEBANQ, where the notes can be called in {{=A(T('between the lines'), _href=URL('hebrew', 'note', vars=dict(id='Mnx2YWxlbmNl', tp='txt_tb1', version='4b', mr='m')))}}.

This is work in progress!

Contributing research

It is also possible to contribute your research to SHEBANQ. As an example, have a look at parallel passages. By means of LAF-Fabric, we have searched the ETCBC data for similar passages. It required a lot of experimentation, for which the IPython notebook is an excellent tool. It has everything: code, documentation, visualization, and it can point to the data. If you have done data analysis on the Hebrew Bible and you want to share your results, the Tools page is where you can do that. In order to be eligible, your notebook must satisfy a few basic conditions:

  1. the results must be based on the ETCBC database of the Hebrew Bible
  2. the results must be reproducible on the basis of the information in the notebook and the data stored with it (we can store additional data at the SHEBANQ site).

Do it yourself
