The creators of the linguistic annotations that drive SHEBANQ, the ETCBC, want to facilitate optimal access to their data by researchers. All operations that are needed for the scholarly process are allowed. Here are the locations of the data, code and documentation. All materials are open source.

In theory, it is possible to build a shebanq website, powered by ETCBC data, on your own server. In practice, it is not completely straightforward. We are working on a complete installation recipe for an Ubuntu server. The document is not yet finished, but if you are interested, here is the current installation document.

Data at DANS
DANS is a national research data archive in the Netherlands. Its repository is called EASY, and there, among many other data sets you find the Hebrew Text Database of the ETCBC.
ETCBC
The ETCBC organization on Github has several repositories related to its Hebrew Text Database.
shebanq-doc
Feature-by-feature documentation of the Hebrew Text Database. Here you find the meaning of the linguistic annotations to the text.
LAF-Fabric
The Hebrew Text Database is archived in the LAF format. LAF-Fabric is a data preprocessing tool that can handle LAF. It is a Python package. On top of it there are packages specialized in the peculiarities of the ETCBC data.
LAF-Fabric notebooks
A recommended way to do data processing with the ETCBC data is through the IPython Notebook. In a notebook you can describe, document and execute Python code. Such notebooks can easily be shared through github and NBviewer. This a repository full of such notebooks, showing a wide range of things that can be accomplished.
DANS-Labs
The DANS-labs organization on Github contains the SHEBANQ project.
SHEBANQ
This is the code that drives the SHEBANQ website. It is based on web2py, whose code is also on Github.
EMDROS
The engine that performs MQL queries against an Emdros text database. Written in C++.
